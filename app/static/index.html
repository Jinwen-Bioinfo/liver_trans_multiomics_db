Help

Follow a demonstrator journey first The quickest serious introduction is to start with the donor quality, injury-versus-rejection, or blood-monitoring walkthrough rather than searching random accessions.
Start with a question Open a use case first if you want the shortest path from a clinical problem to the supporting public datasets and caveats.
Then inspect a study Each study detail view exposes sample composition, downloads, metadata provenance, analysis provenance, and the current claim boundary.
Use feature search as the cross-omics entry Search genes like CXCL10, CYP3A4, COL1A1, VWF, or LST1 to see bulk RNA, single-cell, and protein evidence together.
Read the caveat before the claim Donor organ selection, longitudinal blood monitoring, single-cell matrix summaries, and proteome reference layers are kept separate so the database stays honest.
Open the side panel deliberately Omics Layers shows what is actually processed. Ingest Queue shows what still needs work. Those are not the same thing.
Downloads are part of the evidence Study detail pages expose sample tables, summaries, differential results, and analysis artifacts directly, so a collaborator can reproduce the visible evidence.
Use the hash URL as a bookmark Routes like #study/GSE243887, #use-case/DONOR_LIVER_QUALITY, and #feature/CYP3A4 reopen the same detail view.
NAR readiness is not a vanity score The readiness panel is meant to surface missing infrastructure, documentation, and value-add, not to pretend the project is further along than it is.
Use the glossary when terms start blending together Evidence grades, direct evidence, reference context, and claim boundaries have specific meanings in this resource and should stay that way.

About

LiverTx-OmicsDB integrates public liver transplantation omics datasets into a curated evidence model. It preserves public accession provenance while adding transplant-specific labels, QC status, and biological summaries.
