# Resource Release and Citation Policy

This document records the current release, citation, and contact posture of LiverTx-OmicsDB.

## Current Release State

- Resource name: `LiverTx-OmicsDB`
- Current version: `0.1.0`
- Current release state: `local_pre_public_prototype`
- Public URL status: `not_yet_public`

At this stage the project is a working local prototype with a functional API and curated evidence model, but it is not yet a stable public web resource.

## How To Cite the Resource Right Now

The resource does **not** yet have a DOI-backed formal release.

Until a release snapshot is minted, the preferred citation posture is:

- cite the project repository
- include the access date
- note that the citation is provisional pending a DOI-backed release

Recommended interim wording:

> LiverTx-OmicsDB project repository, accessed via the public GitHub source until a DOI-backed release is minted.

## Planned Citation Upgrade

Before NAR-facing release, the project should add:

1. a DOI-backed release snapshot
2. a stable citation text
3. a release archive policy
4. a visible version history page

## Release Priorities Before Public Launch

The next release-facing priorities are:

1. public deployment with a stable reviewable URL
2. DOI-backed citation and release snapshot
3. user-facing QC and provenance summary pages
4. contact, maintenance, and update policy page

## Why This Matters

For a database resource, release and citation policy are part of the product. Users and reviewers need to know:

- what version they are looking at
- whether the URL is stable
- how to cite the resource
- how to report issues
- whether the project has an update path beyond a one-off prototype
